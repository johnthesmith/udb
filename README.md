# Концепция обменной единицы

Представляется концепция обменной единицы на основе эквивалентности товара и 
денежной массы по формуле:
```
товар = деньги
``` 

Документ описывает понятия, механизмы реализации и метематическую модель.

В основу заложены идеи принципиального исключения влияния негативных 
экономических явлений, таких как инфляция, дефляция, и тд. Модель обеспечивает 
прозрачную [эмиссиию](./emission.md), контроль резервов и вознаграждений.



## Средство

Концепция реализуется как публичная [база данных UDB](./udb.md), учитывающая 
[товар](./goods.md) и оборот единиц обмена [U](./u.md) на основе информации о товаре.



## Процесс

[Пользователи](./user.md) регистрируют в [UDB](./udb.md) товар, при этом эмитируются 
[единицы U](./u.md), которые одновременно могут использвоаться при обмене и 
отражают состояние экономики. Все [U](./u.md) распределяются между публичным 
[резервом](./reserv.md) для обесечения стабильности и приватным 
[оборотом](./circulation.md) для обмена между [пользователям](./user.md). Для контроля 
и соблюдения правил [UDB](./udb.md) ведется реестр [нарушений](./violation.md).



## Ценность U

[U](./u.md) эмитированы на основе зарегистрированного в UDB 
[товара](./goods.md). Правила и публичность прямо исключают возможность 
неконтролируемой эмиссии, а потому поведение поведение [U](./u.md) предсказуемо 
при позитивных и негативных сценариях. Предсказуемосять - один из ключевых 
параметров для доверия к любой денежной единице. Именно это определит 
самоценость [u](./u.md) за пределами [UDB](./udb.md).
