# Вступление

Представляется концепция обменной единицы на основе эквивалентности товара и 
денежной массы по формуле `товар == деньги`. Документ описывает понятия, 
механизмы реализации и метематическую модель.

В основу заложены идеи принципиального исключения влияния негативных 
экономических явлений, таких как инфляция, дефляция, и тд. Модель обеспечивает 
прозрачную эмиссиию, контроль резервов и вознаграждений.



# Средство

Концепция реализуется как публичная [база данных UDB](./udb), учитывающая 
[товар](./goods) и оборот единиц обмена [U](./u) на основе информации о товаре.



# Процесс

[Пользователи](./user) регистрируют в [UDB](./udb) товар, при этом эмитируются 
[единицы U](./u), которые одновременно могут использвоаться при обмене и 
отражают состояние экономики. Все [U](./u) распределяются между публичным 
[резервом](./reserv) для обесечения стабильности и приватным 
[оборотом](./circulation) для обмена между [пользователям](./user). Для контроля 
и соблюдения правил [UDB](./udb) ведется реестр [нарушений](./violation).



# Ценность U

[U](./u) отражают общую стоимость зарегистрированного в UDB [товара](./goods) и, 
при эмиссии, явно привязанной к товару, может рассматриваться самоценным 
эквивалентом обмена за пределами [UDB](./udb).
