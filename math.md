# Базовая математическая модель

1. Исходим из идеальной эквивалетности денег в [балансе B](./balance.md) и 
[товара G](./balance.md).

```
   B = G    
```

1. [баланс B](./balance.md) поддерживается через конроль 
[эмисиии](./emission.md), c учетом эквивалентности денег и [товара G](./goods.md) 
с поправкой на [нормировочный коэффициент эмиссии ke](./ke.md), который позволяет
компенсировать естественные флуктуации:

```
    B = G * ke
```



2. Внутри [UDB](./udb.md) [Баланс](./balance.md) формируется как сумма 
[резерва](./reserv.md) и [оборота](./circulation.md).

```
    B = R + C
```



3. В свою очередь [резерв](./reserve.md) с [оборотом](./circulation.md) связаны 
через [Целевой коэффициент резерва](./k_et) и для идеальной ситуации выражается 
как:
```
    C = R * k_et
```
таким образом отражается заранее определенное распределение денег.



4. Для автокоррекции долей [оборота](circulation.md) и [резерва](reserve.md) из 
за свободы вывода [товара](./goods.md) в [udb](./udb.md) накапливаются 
отклонения и используется механизм автокоррекции через 
[Эмиссионный коэффициент резерва](./k_er.md):
```
    C = R * k_er
```
при том что [Эмиссионный коэффициент резерва](./k_er.md) стремится к 
[Целевому коэффициенту резерва](./k_et):
```
    lim (k_er → k_tr)
```

5. Общая схема балансировки денег сводится к двум уровням:
    - эквивалентность денег и товара: 
        ```
        B = G * ke
        ```
    - распределение внутри системы: 
        ```
        B = R + C, C = R * k_er, lim (k_er → k_et)
        ```
