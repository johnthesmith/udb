# Нормировочный коэффициент эмиссии

1. Нормировочный коэффициент эмиссии `ke` — положительное скалярное значение с 
   плавающей точкой, используемое для корректировки суммы [эмиссии](./emission.md)
   для [товара](./goods.md) в [UDB](./udb.md).
2. Применяется для корректировки объёма эмитируемых [U](./u.md) на различных этапах жизни товара.
3. `ke` обязательно публичен: [пользователь](./user.md) должен быть информирован о 
его текущем значении до регистрации товара.
4. Рекомендуемое значение по умолчанию — `1.0`.



# Использование

1. ke служит для компенсации системных флуктуаций: если в UDB образовался 
избыток или дефицит U по сравнению с эквивалентом товара, коэффициент 
корректирует эмиссию, чтобы восстановить баланс и поддерживать стабильность 
системы.



# Отклонение k

1. Случаи отклонения `ke` от умолчательного значения демонстрируют намерения 
[UDB](./udb.md) скорректировать баланс.
2. `ke` отражает реакцию системы на флуктуации:
   - `ke < 1.0` снижает эмиссию для компенсации избытка U (дефляции.md),
   - `ke > 1.0` увеличивает эмиссию для восполнения дефицита U (инфляции.md).
